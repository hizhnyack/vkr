# Генерация видео по текстовому промпту (T2V Pipeline)

Веб-приложение для генерации видео 10–15 секунд по текстовому описанию на русском языке. Пайплайн: перевод RU→EN, разбиение на сегменты, генерация через выбранную T2V-модель, сшивка в одно видео.

## Требования

- Python 3.10+
- ОС: Ubuntu 22.04 (рекомендуется)
- Для реальных моделей: GPU с 8 GB VRAM (например RTX 3080). Режим «Заглушка» работает без GPU.
- **ffmpeg** — для надёжной сшивки видео. Установка:
  ```bash
  sudo apt update && sudo apt install ffmpeg
  ```
  Без ffmpeg сшивка выполняется через imageio (может не поддерживать часть кодеков).

## Установка

1. Клонируйте или скопируйте проект в каталог, перейдите в него.

2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python3 -m venv venv
   source venv/bin/activate   # Linux/macOS
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. (Опционально) Для GPU с CUDA установите PyTorch с поддержкой CUDA:
   ```bash
   pip install torch --index-url https://download.pytorch.org/whl/cu118
   ```

## Запуск

```bash
source venv/bin/activate
python app.py
```

Откройте в браузере: http://127.0.0.1:5000

## Использование

1. Введите описание сценария на русском в поле «Промпт».
2. Выберите модель (от лёгкой к тяжёлой). На слабом железе начните с «Заглушка» или «Allegro 360P».
3. Нажмите «Сгенерировать».
4. Наблюдайте статус и лог; по завершении появится плеер и кнопка «Скачать видео».

Переключатель моделей позволяет проверить, какие модели потянут на вашем GPU (в т.ч. RTX 3080 8 GB). При ошибке загрузки или нехватке VRAM для Allegro 720P / Wan приложение подставит заглушку и продолжит сшивку.

## Модели и веса

- **Заглушка** — тестовое чёрное видео без нейросети, веса не нужны.
- **Allegro 360P / 720P** — через Hugging Face Diffusers. При первом запуске веса скачиваются автоматически (Hugging Face Hub):
  - `rhymes-ai/Allegro-T2V-40x360P`
  - `rhymes-ai/Allegro-T2V-40x720P`
- **Wan 2.1 / Wan 2.2** — в стандартной поставке не подключаются (нет соответствующих классов в diffusers). Для их использования нужна установка репозиториев Wan-Video и интеграция в `pipeline.py` по официальной документации:
  - [Wan 2.1](https://github.com/Wan-Video/Wan2.1)
  - [Wan 2.2](https://github.com/Wan-Video/Wan2.2)
  - Модели на Hugging Face: `Wan-AI/Wan2.1-T2V-1.3B`, `Wan-AI/Wan2.2-TI2V-5B`

При отсутствии установленной модели Wan в интерфейсе будет использована заглушка с записью в лог.

## Структура проекта

- `app.py` — Flask: маршруты, API, фоновый воркер.
- `pipeline.py` — перевод, разбиение на сегменты, адаптеры T2V, сшивка.
- `templates/index.html` — единственная страница веб-интерфейса.
- `static/app.css`, `static/app.js` — стили и логика фронтенда.
- `bootstrap-5.0.2-dist/` — Bootstrap 5.0.2 для UI.
- `outputs/` — каталог сгенерированных видео (по task_id).

## API

- `GET /` — страница с формой.
- `GET /api/models` — список моделей (id, name, description).
- `POST /api/generate` — тело `{ "prompt": "...", "model_id": "..." }`, ответ `{ "task_id": "..." }`.
- `GET /api/status?task_id=...` — `{ "status", "progress", "video_url"? }`.
- `GET /api/log?task_id=...` — `{ "lines": ["..."] }`.
- `GET /api/video/<task_id>` — скачивание готового видео.
